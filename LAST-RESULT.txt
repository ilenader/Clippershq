TASK 2: Email notifications for clippers
-------------------------------------------
File: src/lib/email.ts — rewritten with new template system

Template design:
- Dark #09090b background, 560px max-width, 16px border-radius
- Header: "CLIPPERS HQ" in accent blue (#2596be), centered, border-bottom
- Body: light text (#e4e4e7) on dark bg, clean typography
- Footer: "© 2026 Clippers HQ — clipershq.com" in muted gray
- Accent blue for amounts, links, highlights

6 email functions:

1. sendWelcomeEmail(email, username)
   Subject: "Welcome to Clippers HQ"
   Trigger: auth.ts createUser event (new user signup)
   Only sends to non-OWNER users (clippers)

2. sendClipApproved(email, campaignName, earnings)
   Subject: "Your clip was approved!"
   Trigger: POST /api/clips/[id]/review (APPROVED action)
   Shows campaign name and current earnings

3. sendClipRejected(email, campaignName, reason?)
   Subject: "Clip update"
   Trigger: POST /api/clips/[id]/review (REJECTED action)
   Shows campaign name and rejection reason

4. sendPayoutApproved(email, amount)
   Subject: "Payout sent!"
   Trigger: POST /api/payouts/[id]/review (PAID action)
   Shows payout amount

5. sendPayoutRejected(email, amount, reason?)
   Subject: "Payout update"
   Trigger: POST /api/payouts/[id]/review (REJECTED action)
   Shows amount and reason

6. sendCallScheduled(email, amount)
   Subject: "Verification call scheduled"
   Trigger: POST /api/calls (owner creates call request)
   Tells clipper to visit Payouts page

Integration points:
- src/lib/auth.ts: welcome email in createUser event
- src/app/api/clips/[id]/review/route.ts: clip approved/rejected emails (updated signatures to include campaignName)
- src/app/api/payouts/[id]/review/route.ts: payout paid/rejected emails (NEW — added import + email sends)
- src/app/api/calls/route.ts: call scheduled email (NEW — added import + email send)

All emails check user.role === "CLIPPER" before sending (no emails to owner/admin).
All email sends are wrapped in try/catch — failures don't crash the API.
Legacy email exports kept for backward compatibility.
