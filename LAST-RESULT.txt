FIX: NextAuth PKCE on Vercel — Summary
========================================

File: src/lib/auth.ts

Added to NextAuth config:

  trustHost: true,
  cookies: {
    pkceCodeVerifier: {
      name: "next-auth.pkce.code_verifier",
      options: {
        httpOnly: true,
        sameSite: "none",
        path: "/",
        secure: true,
      },
    },
  },

Why: On Vercel serverless, the PKCE code verifier cookie set during the
Discord OAuth redirect wasn't surviving to the callback. The cookie's
default sameSite="lax" gets dropped on cross-origin redirects in some
browsers. Setting sameSite="none" + secure=true fixes this.

trustHost: true tells NextAuth to trust the Host header from Vercel's
proxy, preventing "untrusted host" errors in production.
